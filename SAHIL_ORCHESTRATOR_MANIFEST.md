## 🎁 What I Built For You

### 1. `kimi_bridge/` — The Execution Layer Your Agents Were Missing

Your Ollama agents generate brilliant text. But they can't:
- ❌ `git clone`, `npm install`, `docker build`
- ❌ Write actual `src/App.tsx` or `api/routes.py`
- ❌ Search live web for "latest Next.js 15 breaking changes"
- ❌ Spawn a `testing-evidence-collector` and get screenshot proof
- ❌ Fix a bug, re-test, and loop until PASS

**The bridge fixes this.**

```
kimi_bridge/
├── __init__.py          # Package entry
└── bridge.py            # KimiAgent + KimiSwarm classes
```

**`KimiAgent`** wraps any of your 152 agents into a tool-executing subagent:
```python
from kimi_bridge import KimiAgent

agent = KimiAgent("frontend", working_dir="./output")
script = agent.run(
    "Build a React landing page with Tailwind",
    context={"architecture": "...", "design_system": "..."}
)
# → Generates a run script that executes via Kimi CLI with Shell, WriteFile, SearchWeb
```

**`KimiSwarm`** orchestrates pipelines with quality gates:
```python
from kimi_bridge import KimiSwarm

swarm = KimiSwarm(working_dir="./output")
swarm.register("pm", KimiAgent("pm"))
swarm.register("frontend", KimiAgent("frontend"))
swarm.register("qa", KimiAgent("evidence-qa"))

# Run dev → QA loop with auto-retry
result = swarm.run_dev_qa_loop(
    dev_key="frontend",
    qa_key="qa",
    task="Implement login page",
    max_retries=3
)
```
